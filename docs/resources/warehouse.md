# kargo_warehouse (Resource)

Manages a Kargo Warehouse within a Kargo Project. Warehouses subscribe to artifact sources, such as container image repositories, Git repositories, and Helm chart repositories, then create Freight from newly discovered versions.

Updating subscriptions changes the Warehouse in place. Changing `project` or `name` requires replacement because those fields identify the Kargo object.

## Example Usage

```terraform
provider "kargo" {}

resource "kargo_project" "example" {
  name = "example-project"
}

resource "kargo_warehouse" "example" {
  project = kargo_project.example.name
  name    = "app"

  subscription {
    image {
      repo_url               = "ghcr.io/example/app"
      semver_constraint      = "^1.0.0"
      tag_selection_strategy = "SemVer"
      platform               = "linux/amd64"
    }
  }

  subscription {
    git {
      repo_url          = "https://github.com/example/app-config.git"
      branch            = "main"
      semver_constraint = "^1.0.0"
    }
  }

  subscription {
    chart {
      repo_url          = "https://charts.example.com"
      name              = "app"
      semver_constraint = "^1.0.0"
    }
  }
}
```

## Subscription Types

Each `subscription` block must contain exactly one of `image`, `git`, or `chart`.

Subscriptions are ordered. Keep the order stable when multiple artifact sources are part of the same Warehouse.

## Import

Import an existing Warehouse with `project/name`:

```shell
terraform import kargo_warehouse.example example-project/app
```
