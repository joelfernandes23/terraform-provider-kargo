# Kargo Provider

The Kargo provider manages [Kargo](https://docs.kargo.io/) continuous delivery resources through Terraform. It talks to the Kargo API using the Connect JSON protocol and currently supports Projects plus read-only Project lookups.

## Authentication

The provider supports two authentication methods:

- **Bearer token** - Recommended for automation. Provide a token obtained outside Terraform.
- **Admin password** - Useful for local development. The provider exchanges the Kargo admin password for a token during configuration.

Set exactly one of `bearer_token` or `admin_password`.

Configuration can be provided directly in the provider block or via environment variables:

| Provider Attribute         | Environment Variable              |
|---------------------------|-----------------------------------|
| `api_url`                 | `KARGO_API_URL`                   |
| `bearer_token`            | `KARGO_BEARER_TOKEN`              |
| `admin_password`          | `KARGO_ADMIN_PASSWORD`            |
| `insecure_skip_tls_verify`| `KARGO_INSECURE_SKIP_TLS_VERIFY`  |

Use `insecure_skip_tls_verify` only for local development or test clusters with self-signed certificates.

## Example Usage

```terraform
terraform {
  required_providers {
    kargo = {
      source  = "joelfernandes23/kargo"
      version = "~> 0.3"
    }
  }

  required_version = ">= 1.5.0"
}

variable "kargo_bearer_token" {
  type      = string
  sensitive = true
}

provider "kargo" {
  api_url      = "https://kargo.example.com"
  bearer_token = var.kargo_bearer_token
}
```

## Local Development Example

For the Kind-based development environment from this repository:

```shell
export KARGO_API_URL="https://localhost:31443"
export KARGO_ADMIN_PASSWORD="admin"
export KARGO_INSECURE_SKIP_TLS_VERIFY="true"
```

```terraform
provider "kargo" {}
```

## Resource Model

Kargo Projects are top-level pipeline containers. Warehouses and Stages are namespace-scoped resources that belong to a project; as those resources are added, they should reference either `kargo_project.<name>.name` or `data.kargo_project.<name>.name`.
